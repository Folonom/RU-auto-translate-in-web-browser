# RU Auto Translate

Расширение для Яндекс Браузера (и любого Chromium: Chrome, Edge, Brave), которое переводит **весь текст на странице на русский** прямо в DOM — посты, комментарии, ленту, всплывающие блоки. Работает там, где другие расширения пасуют: **X (Twitter), Reddit и любые сайты с открытым Shadow DOM**.

Перевод с **любого языка** на русский. Бесплатно, без ключей, без сторонних сервисов — используются публичные эндпоинты Google Translate.

## Что умеет

- **Автоперевод страниц на лету** — текст заменяется на русский сразу, как только он появляется (лента X, бесконечный скролл Reddit, динамические подгрузки).
- **Shadow DOM** — заходит внутрь открытых shadow-корней (компоненты `shreddit-*` на новом Reddit, веб-компоненты на других сайтах). Это главная причина, почему другие расширения не переводят посты и комменты Reddit.
- **Перевод выделения** — выделил любой текст → рядом всплывает окошко с переводом. Можно скопировать.
- **Повторный перевод** — если React/фреймворк переписывает текст узла, расширение замечает и переводит заново.
- **Восстановление оригинала** — кнопка «Оригинал» возвращает исходный текст.
- **Горячие клавиши**: `Alt+Shift+R` — перевести страницу, `Alt+Shift+T` — вкл/выкл.
- **Бейдж «RU»** на иконке, когда страница переведена.
- **Не трогает**: русский текст, код, URL, хэндлы `@user`, числа, скрытые элементы, поля ввода.

## Установка в Яндекс Браузер

1. Открой `browser://extensions/` (или `chrome://extensions/`).
2. Включи **«Режим разработчика»** (переключатель в правом верхнем углу).
3. Нажми **«Загрузить распакованное расширение»**.
4. Выбери папку `ru-auto-translate` (где лежит `manifest.json`).
5. Готово. Иконка «RU» появится на панели. Расширение работает сразу — открой любой англоязычный пост на X или Reddit.

> После загрузки обнови уже открытые вкладки (`Ctrl+R`), чтобы content-скрипт внедрился.

## Настройка

Клик по иконке открывает popup:
- **Расширение включено** — главный выключатель.
- **Автоперевод страниц** — перевод ленты/постов/комментариев на лету.
- **Перевод выделения** — всплывающее окошко при выделении текста.
- **Показывать всплывашку** — вкл/выкл окошка перевода выделения.
- **Перевести страницу** / **Оригинал** — ручной перевод и возврат исходника.

## Файлы

- `manifest.json` — конфигурация MV3, права, горячие клавиши.
- `background.js` — service worker: кэш переводов, обращение к Google Translate, бейдж, горячие клавиши.
- `content.js` — внедряется на каждую страницу: обход DOM + Shadow DOM, MutationObserver, очередь перевода, всплывашка выделения.
- `popup.html` / `popup.js` — интерфейс настроек.
- `overlay.css` — стили всплывающего окошка перевода.
- `icon16/32/48/128.png` — иконки.

## Как устроен перевод

1. Content-скрипт обходит DOM, включая открытые shadow-корни, и собирает текстовые узлы (пропуская русский, код, URL и т.п.).
2. Узлы ставятся в очередь, батчатся (до 25 текстов / 2000 символов за запрос) и отправляются в background.
3. Background дедуплицирует, сверяется с кэшем (LRU, 5000 записей) и зовёт `clients5.google.com/translate_a/t` (клиент `dict-chrome-ex`). При ошибке падает на поэлементный резервный эндпоинт `translate.googleapis.com/translate_a/single` (`gtx`).
4. Перевод пишется обратно в тот же текстовый узел; оригинал сохраняется для восстановления.
5. `MutationObserver` ловит появление нового контента (новые твиты, подгруженные комменты) и изменения текста — и всё это тоже переводится.

## Ограничения

- Закрытые shadow DOM (режкая вещь) недоступны — но Reddit/X/большинство сайтов используют открытые.
- Используются бесплатные публичные эндпоинты Google: при очень больших объёмах или блокировке по IP возможны задержки. Расширение устойчиво — переключается на резервный эндпоинт и кэширует повторы.
- Переводятся текстовые узлы; инлайн-атрибуты типа `aria-label`, `alt` намеренно не трогаются, чтобы не ломать разметку.
